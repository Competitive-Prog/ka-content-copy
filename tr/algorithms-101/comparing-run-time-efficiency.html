Kullandığımız algoritmaların çalışma süresi verimliliğini anlamak önemlidir. Bir bilgisayarın bir sorunu çözmesi çok uzun sürdüğünde, daha fazla paraya mal olur ve bilgisayarı diğer değerli sorunlara harcamak için zaman kaybetmez. Ayrıca, algoritma kullanıcıya yönelik bir uygulamada kullanılıyorsa, uygulamadan tamamen vazgeçen hayal kırıklığına uğramış kullanıcılara neden olabilir.

Çalışma sürelerini kategorilere ayırma

Bir algoritmanın çalışma süresini, giriş boyutu arttıkça adım sayısının nasıl arttığına göre kategorize edebiliriz. Her zaman aynı miktarda zaman alıyor mu? Bu sürekli bir artış, çok hızlı bir çalışma süresi. Her zaman girdinin olası her permütasyonuna bakmayı gerektirir mi? Bu üstel bir artış, çok yavaş bir çalışma süresi. Çoğu çalışma süresi aralarında bir yerdedir.

Sabit zaman

Bir algoritma sabit zamanda çalıştığında, giriş boyutu ne kadar büyük olursa olsun, her zaman sabit sayıda adım attığı anlamına gelir.

Logaritmik zaman

Bir algoritma logaritmik zamanda çalıştığında, giriş boyutunun logaritması ile orantılı olarak artar.
İkili arama algoritması, logaritmik zamanda çalışan bir algoritmadır. Algoritmanın daha uzun bir açıklaması için verimlilik ölçümü makalesini okuyun.
İşte sözde kod:
PROCEDURE searchList(numbers, targetNumber) {
  minIndex ← 1
  maxIndex ← LENGTH(numbers)
  REPEAT UNTIL (minIndex > maxIndex) {
    middleIndex ← FLOOR((minIndex+maxIndex)/2)
    IF (targetNumber = numbers[middleIndex]) {
      RETURN middleIndex
    } ELSE {
       IF (targetNumber > numbers[middleIndex]) {
         minIndex ← middleIndex + 1
       } ELSE {
         maxIndex ← middleIndex - 1
       }
     }
  }
  RETURN -1
}
Algoritma, listedeki birden çok öğeye bakmak için bir döngü kullanır, ancak en önemlisi, listedeki her öğeye bakmaz. Özellikle, log 2 n\log_2{n}lo g 2 n log , start base, 2, end base, n items, where n n n n , listedeki öğelerin sayısıdır.

Doğrusal zaman
Bir algoritma doğrusal zamanda büyüdüğünde, adım sayısı giriş boyutuyla doğru orantılı olarak artar.
Uygun şekilde adlandırılmış doğrusal arama algoritması doğrusal zamanda çalışır.
Sözde kod, ikili aramaya kıyasla basitliğini gösterir:
PROCEDURE searchList(numbers, targetNumber) {
  index ← 1
  REPEAT UNTIL (index > LENGTH(numbers)) {
    IF (numbers[index] = targetNumber) {
      RETURN index
    }
    index ← index + 1
  }
  RETURN -1
}
Bu kez, döngü listedeki her öğeye bakar. Bu kapsamlı arama, sıralanmamış bir listedeki öğeleri aramak için gereklidir, çünkü belirli bir öğenin nerede olabileceğini daraltmanın bir yolu yoktur. Bu algoritma her zaman en az listedeki öğeler kadar adım gerektirir.

Kuadratik zamanlı

Bir algoritma ikinci dereceden zamanda büyüdüğünde, adımları kare giriş boyutuna orantılı olarak artar.
Birkaç liste sıralama algoritması, seçim sıralaması gibi ikinci dereceden bir zamanda çalışır. Bu algoritma listenin önünden başlar, ardından listedeki bir sonraki en küçük değeri bulmaya ve geçerli değerle değiştirmeye devam eder.
İşte seçim sıralaması için sözde kod:
i ← 1
REPEAT UNTIL (i > LENGTH(numbers)) {
  minIndex ← i
  j ← i + 1
  // Find next smallest value
  REPEAT UNTIL (j > LENGTH(numbers)) {
    IF (numbers[j] < numbers[minIndex]) {
      minIndex ← j
    }
  }
  // Swap if new minimum found
  IF (minIndex != i) {
    tempNum ← numbers[minIndex]
    numbers[i] ← tempNum
    numbers[minIndex] <- tempNum
  }
}
Algoritma hakkında dikkat edilmesi gereken önemli şey, iç içe geçmiş döngüdür: listedeki her öğe arasında döngü yapar ve bu öğelerin her biri için kalan öğeler arasında tekrar döngü yapar. Polinom vs süperpolinom

Bilgisayar bilimcileri genellikle çalışma sürelerini iki sınıfa ayırır:
  • Polinom zamanı, n k n^k n k n, başlangıç üst simgesi, k, bitiş üst simgesindendaha hızlı artmayan ve sabit zamanı (n 0 n^ 0 n 0 n, başlangıç üst simgesi, 0, bitiş üst simgesi ), logaritmik zaman (log 2 n\log_2{n}lo g 2 n log , start base, 2, end base, n), doğrusal zaman ( n 1 n^1 n1 n, başlangıç üst simgesi, 1, bitiş üst simgesi), ikinci dereceden zaman (n 2 n^2 n 2 n , kare ) ve diğer yüksek dereceli polinomlar (n 3 n ^3 n3 n , küp gibi ).

Bir süperpolinom çalışma süresi, nispeten küçük girdi boyutları için bile, genellikle evrende mevcut olandan daha fazla zaman gerektirir.
Bu yüzden polinom çalışma sürelerini makul ve süperpolinom zamanlarını mantıksız olarak düşünüyoruz. Bir polinom çalışma süresi her zaman ideal değildir (ve genellikle bu süreleri iyileştirmeye çalışırız), ancak en azından uygulanabilir.
Bilgisayar bilimcileri, şu anda süperpolinom zaman gerektiren problemler için polinom zaman algoritmaları bulma çabalarını yoğunlaştırıyorlar, çünkü farkın en önemli olduğu yer burası.
