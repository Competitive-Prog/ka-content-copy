Bilgisayar programları yazarken mantıksal eşdeğerliği anlamak önemlidir, böylece programınızın davranışı hem mantıksal olarak doğru hem de diğer programcılar tarafından anlaşılabilir olur. Mantıksal olarak eşdeğer kod yazmanın birçok yolu vardır, ancak genellikle diğerlerinden daha net ve daha özlü olan bir yol vardır.
Mantıksal eşdeğerlik kuralları ve alternatif programlama yapıları kullanılarak basitleştirilebilen sözde koddaki bazı mantıksal akışlara bakalım.

Yapısal eşdeğerlik

İlk olarak, bir şeyin sık kullanıldığında rapor veren bir değişken ayarlamaya çalıştığımız bir örneği ele alalım. Aşağıdaki gibi bir kod yazabiliriz:
if (numUsers > 100) {
  isFrequentlyUsed ← true
} else {
  isFrequentlyUsed ← false
}
Bu tamamen makul ve net bir kod gibi görünüyor. Bununla birlikte, zeki gözlemci, if/else öğesinin isFrequentlyUsedher bloğunda yapılan tek işlemin . Bu değeri şu şekilde ayarlar:
  • trueise numUsers > 100true
  • false diğer durumda (nerede numUsers > 100 false)
if/else bloğunun tamamını tek bir kod satırıyla değiştirebiliriz:
isFrequentlyUsed ← numUsers > 100
Bu, bir numUsers > 100boolean (true veya false) üretecek ve ardından değeri doğrudan değişkene atayacaktır. isFrequentlyUsed
Bu kuralı genelleştirmek için, bu formda bir if/else varsa:
if (E) {
  V ← true
} else {
  V ← false
}
Bu ifade mantıksal olarak eşdeğerdir:
V ← E

Olumsuzlama ile yapısal eşdeğerlik

Benzer ancak zıt bir işlem yapan, nadiren bir şey kullanılıyorsa rapor veren aşağıdaki kodu göz önünde bulundurun:
if (numUsers > 100) {
  isRarelyUsed ← false
} else {
  isRarelyUsed ← true
}
İlk başta, sıkışmış gibi görünüyoruz çünkü şimdi kodumuz yukarıdakilerin tam tersini yapıyor. Değeri şu şekilde ayarlar:
  • falsetrue if numUsers > 100 (aynı değer değil!)
  • true aksi takdirde (ne zaman numUsers > 100 false)
Neyse ki, eşdeğer basit booleanlar hakkındaki ilk makaledeki mantıksal eşdeğerlik kuralını kullanabilir ve bir Boolean'ın zıt değerini elde etmek için kullanabiliriz NOT .
Yani yukarıdaki if/else bu kod satırıyla aynıdır:
isRarelyUsed ← NOT (numUsers > 100)
Genel olarak kuralımız şudur:
if (E) {
  V ← false
} else {
  V ← true
}
Şunlarla değiştirilebilir:
V ← NOT (E)
