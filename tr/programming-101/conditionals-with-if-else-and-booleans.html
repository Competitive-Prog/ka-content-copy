if / else ve Booleans | ile koşullular AP CSP (makale) | Khan Akademisi

İnsanlar olarak, öğle yemeğinde ne yiyeceğimiz veya yağmurluk giyip giymeyeceğimiz gibi her gün kararlar alıyoruz. Bu kararları verirken, öğle yemeği kararı için buzdolabımızın içeriği veya yağmurluk kararı için dışarıdaki hava durumu gibi dünyamızın birçok koşulunu göz önünde bulundururuz.
Bilgisayar programları ayrıca koşulluların (if/else) içindeki Boole ifadelerini (true/false) kullanarak kararlar verir. Koşullular sayesinde, programlar farklı girdilere ve parametrelere göre farklı yanıt verebilir.

Basit bir koşullu

Her gün dışarıda ne giyeceğimize karar vermemize yardımcı olacak bir program yazdığımızı hayal edin. Bu programla bulabileceğimiz birçok koşul var, ancak basit başlayalım: sıcaklık donma noktasının altındaysa, program bizi kar ceketi giymemiz konusunda uyarmalıdır.
İşte bir akış şemasındaki bu mantık:
"Sıcaklık 32'den küçük mü yoksa ona eşit mi?" yazan elmas ile başlayan akış şeması. Diamond, "Ekran: "Kar ceketi giy!" yazan dikdörtgene yol açan "TRUE" etiketli oka sahiptir.
[Bir akış şemasını nasıl okursunuz?]
Bu akışı JavaScript'te bir if ifadesi kullanarak programlayabiliriz:
if (temperature < 32) {
  println("Wear a snow jacket!");
}
Bu sözdizimini parçalayalım. Her if ifadesinin genel yapısı şöyle görünür:
if (<condition>) {
  <instructions>
}
Durum ve talimatlar farklı olan şeydir; sihrin gerçekleştiği yer burasıdır.
Koşul bir Boole ifadesidir: ya falseda true . Boole değerleri, programlama dillerinde başka bir veri türüdür ve yalnızca true veya false'u tutabilirler.
[Bitleri hatırlıyor musun?]
Örneğin, bu Boole ifadesinin sonuçlarını bir değişkende depolayabiliriz:
var belowFreezing = temperature < 32;
Geçerli değerine temperaturebağlı olarak, belowFreezing değişken şimdi ya falseda true .
Koşulu değişkenle bile değiştirebiliriz:
var belowFreezing = temperature < 32;
if (belowFreezing) {
  println("Wear a snow jacket!");
}
Nasıl kodlarsak, koşulun ya da true false, çünkü bu, bilgisayara kıvırcık parantezlerin içindeki talimatları uygulayıp uygulamayacağına dair bir değerlendirme yapması gerekir.
Talimatlar, programınızın yapabileceği herhangi bir şey olabilir ve gerektiği kadar uzun olabilir.
✏️ Aşağıdaki programdaki değeri temperature değiştirmeyi deneyin ve ne olduğunu görün. Koşulu veya talimatları da değiştirebilirsiniz.
var sıcaklık = 30;
if (sıcaklık < 32) {
    println("BRRR dışarıdaki soğuğu!");
    println("Kardan adam yapmak ister misin?");
    println("☃☃☃☃☃☃☃☃");
    println("Ya da biraz kar tanesi suşisi yiyin?");
    println(" ❆ ❆ ❄ ❄ ❄ ❅ ❄ "❄);}

Karşılaştırma işleçleri

Yukarıdaki Boole ifademiz, bir sayının başka bir sayıdan küçük veya ona eşit olup olmadığını belirlemek için karşılaştırma işlecini <= kullandı. Programlama dillerinde değerler arasındaki farklı ilişkileri, özellikle sayıları belirlemek için birçok karşılaştırma işleci vardır.
JavaScript'teki ilişkisel işleçler şunlardır:
İşleçAnlamıDoğru ifadeler   \,
>Daha büyük60 > 32
>=Büyük veya eşit60 >= 3232 >= 32
<Daha az20 < 32
<=Daha küçük veya eşit20 <= 3232 <= 32
===Sıkı eşitlik32 === 32
==Eşitlik32 == 32"32" == 32
!==Katı eşitsizlik30 !== 32"32" !== 32
!=Eşitsizlik30 != 32
JavaScript burada biraz zor bir dildir, çünkü hem eşitlik işleçlerine hem de "katı" eşitlik işleçlerine sahiptir.
Eşitlik işleçleri önce ifadenin her iki tarafını da aynı türde olmaya zorlamaya çalışacak (her ikisini de sayılara dönüştürerek olduğu gibi) ve ardından eşitliği denetleyecektir. Katı eşitlik operatörü, iki değerin farklı türler olduğunu görecek ve hemen birbirlerine eşit olamayacaklarına karar verecektir. Daha öngörülebilir kodlara yol açtıkları için katı işleçleri kullanmanızı öneririz.
Çoğu programlama dilinde her iki tür eşitlik işleci de yoktur, ancak dillerin iki değişkenin eşit olup olmadığına karar verme biçiminde başka farklılıklarla karşılaşabilirsiniz.
✏️ Aşağıdaki program, değişkene göre yourAge her türlü mesajı görüntüler. Hangi çıktının görüntülendiğini görmek için bunu farklı değerlerle değiştirmeyi deneyin. Programa daha fazla if deyim de ekleyebilirsiniz.

if (yourAge >= 0) {
    println("Tebrikler, doğdunuz!");
}

if (yourAge === 0) {
    println("Sen sadece bir bebeksin!");
}

if (yourAge < 3) {
    println("Henüz konuşamıyorsun bile! Nasıl kodlayabilirsiniz?");
}

if (yourAge === 5) {
    println("Okulun ilk gününe hazır ol!");
}

if (yourAge >= 16) {
    println("Araba kullanabilirsiniz! Belki! Eyalet yasalarınızı kontrol edin.");
}

if (yourAge >= 18) {
    println("Oy kullanabilirsiniz!");
}

if (yourAge >= 21) {
    println("Kamu görevi için uygunsunuz!");
}

if (yourAge >= 35) {
    println("Şimdi başkanlığa aday olabilirsiniz!");
}

if (yourAge >= 65) {
    println("Şimdi emekli olabilirsiniz!");

Else ifadesi

if ifadeleriyle, programlarımız yalnızca koşul doğruysa bir dizi talimatı yürütebilir. Programlarımızın, koşul yanlış olduğunda farklı bir talimat kümesi yürütmesini istiyorsak, else bir deyim kullanabiliriz.
Süper basit bir şifre kontrol programı hayal edin. Mantık akışı şöyle görünür:
"Parola TOPSECRET'a eşit mi?" yazan elmas ile başlayan akış şeması. İlk ok "TRUE" olarak etiketlenir ve elmastan "Ekran: 'Anladın!'" yazan dikdörtgene götürür. İkinci ok "YANLIŞ" olarak etiketlenir ve elmastan "Görüntüle: 'Yeniden dene!'" yazan dikdörtgene götürür.
Bunu bir ifadeyi ve ardından else bir if ifadeyi kullanarak programlayabiliriz:
if (password === "TOPSECRET") {
   println("You got it!");
} else {
   println("Try again!");
}

Sözde koddaki koşullular

[Sözde kod nedir?]
Bu sözde kod tek if bir ifadeyi temsil eder:
IF (<condition>)
{
     <instructions>
Bu sözde kod, bunun yerine yürütülebilecek ikinci bir yönerge kümesi içeren bir ifade içerir else :
IF (<condition>)
{
     <instructions 1>
} 
ELSE 
{
     <instructions 2>
}
Bunlar, sözde koddaki ilişkisel işleçlerdir:
OperatörAnlamı
a = bEşitlik
a ≠ bEşitsizlik
a > bDaha büyük
a < bDaha az
a ≥ bBüyük veya eşit
a ≤ bDaha küçük veya eşit
İşte sözde koddaki parola denetleyicisi mantığı:
IF (password = "TOPSECRET")
{
   DISPLAY("You got it!")
}
ELSE {
   DISPLAY("Try again!");
}
