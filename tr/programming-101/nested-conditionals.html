İç içe koşullular (if/else/if) | AP CSP (makale) | Khan Akademisi

Bilgisayar programları, bir programın aşağı inmesi için doğru yolu seçmek üzere koşulluları kullanır. Bir program yalnızca iki yoldan birini seçtiğinde, basit bir koşullu (if/else) kullanabilir. Bir program birçok yoldan birini seçtiğinde, iç içe geçmiş veya zincirlenmiş koşullular kullanabilir.

İç içe koşullular

Bir sayının pozitif, negatif veya sıfır olup olmadığını bildiren bir program düşünün. Bu programın 3 yol arasından seçim yapması gerekir.
Mantığın akış şeması şöyle görünebilir:
2 elmas, 3 dikdörtgen ve 4 oklu akış şeması.
  • Üst elmas "Sayı 0'dan büyük mü?" sorusunu içerir.
  • Üst elmastan, doğru olarak işaretlenmiş bir ok, "işaret pozitif" metniyle dikdörtgene yol açar.
  • Üst elmastan, yanlış işaretli bir ok, "Sayı 0'dan küçük mü?" sorusuyla elmasa yol açar.
  • İkinci elmastan, doğru olarak işaretlenmiş bir ok, "işaret negatif" metniyle dikdörtgene yol açar.
  • İkinci elmastan, yanlış işaretli bir ok, "işaret nötrdür" metniyle dikdörtgene yol açar.
[Bir akış şemasını nasıl okursunuz?]
Bunu iç içe koşullular kullanarak JavaScript'e çevirebiliriz. İşte eşdeğer JS:
var numberSign;

if (number > 0) {
  numberSign = "positive";
} else {
  if (number < 0) {
    numberSign = "negative";
  } else {
    numberSign = "neutral";
  }
}

Dış koşulumuz, akış şemasındaki üst elmasa karşılık gelen basit ifbir /else'dir. Sonra diğerinin içinde, akış şemasındaki ikinci elmasa karşılık gelen bir iç koşul yerleştiririz.
JavaScript gibi metin tabanlı dillerde, iç içe geçmiş koşullularınızın sözdizimine ve biçimlendirmesine dikkat etmeniz önemlidir. Her talimat kümesinin etrafında kıvırcık parantezleri nasıl kullandığıma dikkat edin ve iç koşulluyu else. Bu, diğer programcıların (gelecekteki ben de dahil!) mantığın akışını daha iyi anlamalarına yardımcı olur.

Alternatif iç içe geçme akışları

İç içe geçmiş şartlıların çoğu için, çeşitli koşulların sırası için birçok olasılık vardır.
Bir sayının pozitif/negatif/nötr olup olmadığına karar vermek için, bu akış şemasında olduğu gibi, sayının sıfıra eşit olup olmadığını sorarak da başlayabiliriz:
2 elmas, 3 dikdörtgen ve 4 oklu akış şeması.
  • Üst elmas "Sayı 0'a eşit mi?" sorusunu içerir.
  • Üst elmastan, doğru olarak işaretlenmiş bir ok, "işaret nötrdür" metniyle dikdörtgene yol açar.
  • Üst elmastan, yanlış işaretli bir ok, "Sayı 0'dan büyük mü?" sorusuyla elmasa yol açar.
  • İkinci elmastan, doğru olarak işaretlenmiş bir ok, "işaret pozitif" metniyle dikdörtgene yol açar.
  • İkinci elmastan, yanlış olarak işaretlenmiş bir ok, "işaret negatif" metniyle dikdörtgene yol açar.
JavaScript'e çevrildiğinde, bu akış şöyle görünür:
var numberSign;

if (number === 0) {
  numberSign = "neutral";
} else {
  if (number > 0) {
    numberSign = "positive";
  } else {
    numberSign = "negative";
  }
}
Bu akış, diğer akış kadar doğrudur ve girişler için ilk akışla aynı değer kümesini çıkarır.
Aslında, aynı sonucu elde etmek için en az dört olası akış daha vardır.
[Daha fazla akışa bakın]
Programınız hangi akışı izlemelidir? Her biri aynı sonucu elde ettiği sürece, en doğal ve sezgisel görünen akışa gidin. Akışı programladıktan sonra, beklendiği gibi çalıştığından emin olmak için programınızı her zaman bir dizi değerle test edin.

Daha derin iç içe geçme

Dünyadaki her şeyin düzgün bir şekilde üç kategoriye ayrılması düzgün olurdu, ama dünya bundan daha dağınık bir yer. Neyse ki, koşulluları gerektiği kadar seviyeyi iç içe geçirebiliriz ve programımız birçok farklı yoldan gidebilir.
Örneğin, bir salsa'yı biberlerinin Scoville derecelendirmesine dayanarak hafif, orta, sıcak veya aşırı olarak sınıflandırmak için bir program yazabiliriz.
Bu akış şeması istenen mantığı açıklar:
3 elmas, 4 dikdörtgen ve 6 ok içeren akış şeması.
  • En iyi elmas "Derecelendirme 5.000'den az mı?" sorusunu içerir.
  • Üst elmastan, doğru olarak işaretlenmiş bir ok, "seviye hafif" metniyle dikdörtgene yol açar.
  • Üst elmastan, yanlış işaretli bir ok, "Derecelendirme 20.000'den az mı?" sorusuyla elmasa yol açar.
  • İkinci elmastan, true olarak işaretlenmiş bir ok, "seviye orta" metniyle dikdörtgene yol açar.
  • İkinci elmastan, yanlış işaretli bir ok, "Derecelendirme 70.000'den az mı?" sorusuyla elmasa yol açar.
  • Üçüncü elmastan, doğru olarak işaretlenmiş bir ok, "seviye sıcak" metniyle dikdörtgene yol açar
  • Üçüncü elmastan, yanlış işaretli bir ok, "seviye aşırı" metniyle dikdörtgene yol açar.
Bunu JavaScript'te daha önce olduğu gibi /else ifadeleriyle, ancak ek bir iç içe geçme seviyesiyle ifuygulayabiliriz.
var level;

    level = "medium";
  } else {
    if (rating < 70000) {

Zincirleme koşullular

Programlama yaparken, kendimizi genellikle tek bir değişkenin olası değerlerini kontrol etmek için koşulluları kullanırken buluruz.
Bunun iki örneğini zaten gördük ve her birini bir tabloda özetleyebiliriz:
koşulçıkışı
number > 0"positive"
number < 0"negative"
number === 0"neutral"
koşulçıkışı
rating < 5000"mild"
rating < 20000"medium"
rating < 70000"hot"
rating >= 70000"extreme"
Birçok programlama dili, bu değerleri kontrol etmenin daha iyi bir yolu olan "zincirlenmiş" koşullular yazmanın bir yolunu içerir.
JavaScript dilinde, ifadeleri kullanarak else if koşulluları zincirleyebiliriz.
Sayı işareti denetleyicisini zincirleme koşullularla nasıl yazabiliriz:
var numberSign;

if (number > 0) {
  numberSign = "positive";
} else if (number < 0) {
  numberSign = "negative";
} else {
  numberSign = "neutral";
}
Benzer kodu şurada da görebilirsiniz: App Lab | Piton
Bu kod orijinal koda çok benzer, ancak içinde bir else if iç içe geçmiş else yerine bir if kullanır.
Zincirlenmiş bir koşullu, özellikle koşulların sayısı arttıkça ve genellikle biraz daha kısakça okunması genellikle daha kolaydır.
İşte zincirleme koşullularla yazılmış salsa seviyesi hesaplayıcısı:
var level;

if (rating < 5000) {
  level = "mild";
} else if (rating < 20000) {
  level = "medium";
} else if (rating < 70000) {
  level = "hot";
} else {
  level = "extreme";
}
Benzer kodu şurada da görebilirsiniz: App Lab | Piton
Bunu iç içe koşullu orijinal sürümle karşılaştırın. Çok daha iyi akar, değil mi?
Tüm diller koşulluları zincirleme yeteneği sağlamaz, ancak özelliği içeren JS veya Python gibi bir dilde programlama yapıyorsanız, bu gibi durumlar için bundan yararlanın.
Sınav İpucu: AP CSP sınavı zincirleme koşullular içermez . Her durumda iç içe koşullular kullanır.

Sözde kodda iç içe koşullular

[Sözde kod nedir?]
Bu sözde kod, bir dış IF/ ve bir iç /ELSEELSE ile basit bir iç IFiçe koşullu gösterir:
IF (<condition 1>)
{
     <instructions 1>
} 
ELSE 
{
    IF (<condition 2>)
    {
         <instructions 2>
    } 
    ELSE 
    {
        <instructions 3>
    }
}
İşte sözde koddaki işaret kontrol programı:
IF (number > 0)
{
    numberSign ← "positive";
}
ELSE
{
    IF (number < 0)
    {
        numberSign ← "negative";
    }
    ELSE
    {
        numberSign ← "neutral";
    }
}
